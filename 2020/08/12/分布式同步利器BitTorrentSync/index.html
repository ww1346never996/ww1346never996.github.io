BT Sync，不只是同步工具

预先告知，该工具正常使用途径已被you know what封杀，但是仍旧可以使用，该教程旨在教授一些该工具的初步使用方法，请勿用于传播非法内容！若因个人使用导致的任何责任，本人概不承担。

简单介绍

相信玩网老司机一定都听说过BT下载吧。BT是一种互联网的P2P传输协议，全名”BitTorrent”。BT Sync 与BT下载一样，同为BItTorrent公司的产品。故BT Sync 也是通过P2P传输的。

BT Sync 主要是作为同步工具来使用的，它能够让你快速的在几个不同的设备直接同步文件。具有增量同步的功能。

由于是P2P的传输方式，我们无需服务器，无需公网ip即可进行同步，只要我们在中心服务器找到节点接入网络，就可以一直享受庞大的资源树。

插播一句个人观点：BT这种方式真的是互联网精神的典范：人人为我，我为人人。可惜的是随着互联网公司的发展，这一点已经给大型商业公司碾进了互联网的角落了。

那么它何以做网盘呢？答案也是P2P。

每个人都是服务器，这就是BT的精神。利用BT Sync，享有私钥的用户可以从节点网络中正在运行的用户那里读取数据，这让其脱离了传统网盘的几个桎梏：

  1. 存储空间限制
  2. 下载流量限制 这也是百度网盘广为诟病的一点
  3. 网络审查及版权问题
  4. 服务费用

BT Sync的使用

这里就是重点中的重点了！使用这个小玩意只需要简单的三步

找一个文件夹

这没啥好讲的，开个属于它的文件夹就行，路径最好别带中文。

接受同步密钥

BT Sync最重要的就是同步密钥。分为读写密钥和只读密钥。顾名思义，读写密钥的持有者拥有读写的权限，能够更新目录中的内容，拥有只读密钥的人无法对同步目录修改，只能读取。你是可以改里面的东西的，但是这个改动不会传给别的用户。

1.4版本里右侧有一个设置按钮

点击输入密钥

把你的接受密钥丢进去，选择你的文件夹，完成。

发起同步

要发起同步，你需要一个用来同步的文件夹废话，你对该文件夹的改动会同步给拥有密钥的人。

点击这个，选择你的文件夹

这是一些设置，不多介绍

在这个三个小点这里点击，可以修改偏好和断开连接不会删除文件夹，只会断开与网络的连接

如何科学使用

以上说了那么多，你可能也开始了使用。然而，由于you know what的封杀，我们无法正常的访问节点网络，也就是说，刚刚搞了半天，最后我们却无法正常使用！这时候，我们就要收回之前卖的那个关子了，祭出我们的杀器——DHT网络。

BT Sync的节点发现

对于任何一个P2P工具，节点发现都是开始运作的第一步。BT Sync实现节点发现，主要有两个途径。

Tracker server

追踪服务器（Tracker server）就是节点之间的中介，通过连接该服务，节点之间可以互相发现，获取信息。而正是因为这个服务被封了，导致我们的BT Sync无法正常使用。

DHT网络

分布式是个啥玩意？三言两语肯定讲不清。所以DHT想讲清楚，肯定不是我这个普及用法的博客的工作。我们大概可以这样描述：有一张网络，上面的每个节点都有这张网络的蓝图，你想要加入这张网络，只需要找到一个在网络中的节点，告诉他你要进来了，这样就能顺藤摸瓜找到整张网络，并加入他们。这个找到第一个节点的过程叫：bootstrap。这一步就是利用DHT网络的关键。

如何在国内进行节点发现

由于我们无法访问Tracker server，而且又不可能有好心人告诉你他也在网络里，并且告诉你怎么连进来。所以科学上网还是免不了的。我们先挂上代理，等待第一次的同步。当客户端界面出现了若干节点数，且已经开始同步，设置DHT网络：

如果有局域网同步需求，需勾选搜索LAN

至此，你应该已经连接进了DHT网络，如果想要测试是否能够免翻使用，请去除勾选中继服务器及跟踪程序服务器。关闭科学上网，重启BT Sync看看节点列表是否还有节点，能否正常同步。若能够正常使用，今后就能免翻一直使用啦！

关于DHT的注意事项

保证经常运行BT Sync，这样才能更上DHT的同步。如果较长时间未使用导致同步失败，就再次按照上面的方法发现节点就行了。
